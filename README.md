# 🪶 ToneCraft

**Draft social posts that sound like _you_.**

ToneCraft learns a writer's voice from a small library of their past posts and
uses **few-shot prompting** to generate fresh posts on any topic — in the same
tone, length, and language they'd naturally use. No fine-tuning, no training
runs: just your own words, fed back to a fast language model as live examples.

Pick a **topic**, a **length**, and a **language**, hit *Generate*, and you get
a post written in your style.

---

## How it works

ToneCraft runs in two clean stages:

```
            STAGE 1 — INGEST (offline, run once)
 data/raw_posts.json ──► tonecraft.ingest ──► data/processed_posts.json
   {text, engagement}     · count lines          + line_count
                          · detect language       + language
                          · extract + unify tags   + tags

            STAGE 2 — GENERATE (live, in the app)
 topic + length + language
        │
        ▼
 StyleLibrary.examples_for(...)  ──►  matching past posts
        │
        ▼
 PostGenerator.build_prompt(...)  ──►  instruction + few-shot examples
        │
        ▼
 tonecraft.llm.complete(...)  ──►  Groq / Llama  ──►  new post
```

- **Stage 1** enriches each raw post with metadata. Line counts are computed
  deterministically; language and topic tags are extracted by the model and
  then merged into a clean canonical vocabulary.
- **Stage 2** selects the past posts that match the requested topic/length/
  language and shows them to the model as examples, so the generated post
  inherits the author's voice.

## Project layout

```
.
├── app.py                     # Streamlit UI (entry point)
├── tonecraft/                 # core package
│   ├── config.py              # central settings (env + defaults)
│   ├── exceptions.py          # typed error hierarchy
│   ├── llm.py                 # Groq client wrapper
│   ├── style_library.py       # loads dataset, serves style examples
│   ├── generator.py           # builds the few-shot prompt, generates posts
│   └── ingest.py              # offline raw → processed pipeline
├── scripts/
│   └── build_dataset.py       # CLI for the ingest pipeline
├── data/
│   ├── raw_posts.json         # source posts {text, engagement}
│   └── processed_posts.json   # enriched posts (generated by ingest)
├── requirements.txt
└── .env.example
```

## Setup

1. **Install dependencies** (a virtual environment is recommended):
   ```bash
   pip install -r requirements.txt
   ```

2. **Add your Groq API key.** Create one at
   https://console.groq.com/keys, then:
   ```bash
   cp .env.example .env
   # edit .env and set GROQ_API_KEY=...
   ```

3. **Run the app:**
   ```bash
   streamlit run app.py
   ```

## Rebuilding the dataset

`data/processed_posts.json` ships ready to use. To regenerate it from
`data/raw_posts.json` (e.g. after adding your own posts):

```bash
python -m scripts.build_dataset            # uses default paths
python -m scripts.build_dataset -v         # verbose logging
```

Add your own posts by appending `{ "text": "...", "engagement": 0 }` objects to
`data/raw_posts.json` and re-running the command above.

## Configuration

All tunables live in `tonecraft/config.py` and can be overridden via environment
variables:

| Variable                | Default                     | Purpose                          |
| ----------------------- | --------------------------- | -------------------------------- |
| `GROQ_API_KEY`          | _(required)_                | Groq API credential              |
| `TONECRAFT_MODEL`       | `llama-3.3-70b-versatile`   | Chat model used for generation   |
| `TONECRAFT_TEMPERATURE` | `0.7`                       | Sampling temperature             |
